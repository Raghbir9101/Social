ISSUE:
  Rows with source "N/A" and IP "::1" (localhost) showed "‚Äî" for location.
  Root cause: when the server receives connections from localhost (::1) it skips
  IP-based lookup. The client's fallback also calls the backend /visitors/location
  endpoint, which hits the same dead end.

FIX ‚Äî client/src/lib/location.js:
  Updated getIPLocation() to:
  1. Try the server endpoint first (works in production where headers carry real IP).
  2. If the server returns source='unknown', fall back to a DIRECT browser-to-ipapi.co
     fetch. Because this request originates from the user's browser, ipapi.co sees the
     user's real public IP ‚Äî bypassing the localhost server-IP problem entirely.
  Note: existing rows already in the DB with no location cannot be retroactively fixed;
  only new visits after this change will have location populated.

FILES: client/src/lib/location.js

[2026-07-09] Fix: Critical Tailwind CSS cascade layer conflict ‚Äî broken layout, padding, margins
---
ISSUE:
  `client/src/index.css` contained a bare `* { margin: 0; padding: 0; box-sizing: border-box; }`
  reset OUTSIDE any CSS `@layer` block. In CSS Cascade Layers, unlayered styles always have
  higher cascade precedence than named-layer styles, regardless of selector specificity.
  Because all Tailwind v4 utility classes live inside `@layer utilities`, this single unlayered
  rule silently overrode EVERY Tailwind margin and padding utility:
    - `lg:ml-64` ‚Üí margin-left: 0 (instead of 256px) ‚Üí sidebar covered the page content
    - `p-4`, `p-5`, `p-6`, `px-3`, `py-2` ‚Üí all padding: 0 ‚Üí no inner spacing in any card
    - `space-y-6`, `mb-1`, `mt-4` ‚Üí all margin: 0 ‚Üí section spacing collapsed
  The result was a completely broken layout: the sidebar overlapped the content, stat cards
  had no internal padding, and all inter-element spacing was gone.
  Tailwind v4's preflight (in `@layer base`) already resets margin and padding ‚Äî the custom
  reset was both redundant and actively destructive.

FIX:
  Removed the `* { margin: 0; padding: 0; box-sizing: border-box; }` block from
  `client/src/index.css`. Tailwind v4's built-in preflight provides the same reset inside
  the proper `@layer base`, which does not conflict with utility classes.

FILES CHANGED:
  - client/src/index.css  (removed redundant unlayered * reset)

RESULT:
  All Tailwind margin, padding, and spacing utilities now apply correctly. The sidebar
  offset (`lg:ml-64`), card padding (`p-4`, `p-5`), section spacing (`space-y-6`), and
  all other layout utilities are restored.

[2026-07-09] Fix: Tailwind / Lightning CSS backdrop-filter bug
---
ISSUE:
  The `.glass` and `.glass-strong` custom CSS classes in `client/src/index.css` defined
  both `-webkit-backdrop-filter` and `backdrop-filter` with multi-function values
  (e.g. `blur(24px) saturate(150%)`). Lightning CSS (the CSS optimizer used by
  Tailwind v4's Vite plugin) was incorrectly retaining only the `-webkit-backdrop-filter`
  version in the compiled output and dropping the standard `backdrop-filter` property.
  Chrome 104+ removed support for `-webkit-backdrop-filter`, so the glassmorphism blur
  effect was silently broken in all modern Chromium-based browsers.
  `.glass-light` (which only used `blur()`) was unaffected ‚Äî confirming the bug was
  triggered specifically by multi-function backdrop-filter values.

FIX:
  Removed the manual `-webkit-backdrop-filter` prefixes from `.glass` and `.glass-strong`
  in `client/src/index.css`. With only the standard `backdrop-filter` in the source,
  Lightning CSS now correctly adds the `-webkit-` prefix via autoprefixing AND retains
  the standard property ‚Äî resulting in both being present in the compiled output.

FILES CHANGED:
  - client/src/index.css  (removed -webkit-backdrop-filter from .glass and .glass-strong)

RESULT:
  Compiled CSS now contains both `backdrop-filter` and `-webkit-backdrop-filter` for
  `.glass` and `.glass-strong`, restoring the glassmorphism blur effect across all
  modern browsers (Chrome, Firefox, Safari).
